# 股票交易管理系统需求文档

## 1. 功能概述

在现有选股系统基础上，增加完整的交易管理功能，包括买入/卖出记录、持仓管理、卖点提示和消息通知。

## 2. 数据表设计

### 2.1 持仓表（stock_position）
```sql
CREATE TABLE stock_position (
    id INT AUTO_INCREMENT PRIMARY KEY,
    ts_code VARCHAR(12) NOT NULL COMMENT '股票代码',
    stock_name VARCHAR(50) COMMENT '股票名称',
    strategy VARCHAR(10) COMMENT '选股策略(b0/b1/b2/b3)',
    screen_date DATE COMMENT '选股日期',
    buy_date DATE NOT NULL COMMENT '买入日期',
    buy_price DECIMAL(10,3) NOT NULL COMMENT '买入均价',
    buy_amount INT NOT NULL COMMENT '买入数量(股)',
    buy_total DECIMAL(12,2) COMMENT '买入总金额',
    current_price DECIMAL(10,3) COMMENT '当前价格',
    current_value DECIMAL(12,2) COMMENT '当前市值',
    profit_loss DECIMAL(12,2) COMMENT '盈亏金额',
    profit_loss_pct DECIMAL(5,2) COMMENT '盈亏百分比',
    status ENUM('holding','sold','partial_sold') DEFAULT 'holding' COMMENT '持仓状态',
    is_alert TINYINT DEFAULT 0 COMMENT '是否触发卖点提示',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    KEY idx_ts_code (ts_code),
    KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='持仓表';
```

### 2.2 交易记录表（stock_trade）
```sql
CREATE TABLE stock_trade (
    id INT AUTO_INCREMENT PRIMARY KEY,
    position_id INT NOT NULL COMMENT '持仓ID',
    ts_code VARCHAR(12) NOT NULL COMMENT '股票代码',
    stock_name VARCHAR(50) COMMENT '股票名称',
    trade_type ENUM('buy','sell','sell_partial') NOT NULL COMMENT '交易类型',
    trade_date DATE NOT NULL COMMENT '交易日期',
    trade_price DECIMAL(10,3) NOT NULL COMMENT '成交价',
    trade_amount INT NOT NULL COMMENT '成交数量(股)',
    trade_total DECIMAL(12,2) NOT NULL COMMENT '成交总金额',
    profit_loss DECIMAL(12,2) COMMENT '盈亏金额',
    notes TEXT COMMENT '备注',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    KEY idx_position (position_id),
    KEY idx_ts_code (ts_code),
    KEY idx_trade_date (trade_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='交易记录表';
```

### 2.3 卖点提示表（stock_alert）
```sql
CREATE TABLE stock_alert (
    id INT AUTO_INCREMENT PRIMARY KEY,
    position_id INT NOT NULL COMMENT '持仓ID',
    ts_code VARCHAR(12) NOT NULL COMMENT '股票代码',
    stock_name VARCHAR(50) COMMENT '股票名称',
    alert_type VARCHAR(20) NOT NULL COMMENT '提示类型(s1/s2/s3/sm1)',
    alert_date DATE NOT NULL COMMENT '触发日期',
    alert_price DECIMAL(10,3) COMMENT '触发价格',
    trigger_desc TEXT COMMENT '触发描述',
    status ENUM('pending','processed','ignored') DEFAULT 'pending' COMMENT '处理状态',
    processed_at TIMESTAMP NULL COMMENT '处理时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    KEY idx_position (position_id),
    KEY idx_status (status),
    KEY idx_alert_date (alert_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='卖点提示表';
```

## 5. 每日追踪逻辑扩展

### 5.1 追踪脚本修改

在现有的 `track_b*_daily.py` 中增加持仓检查逻辑：

```python
def check_position_signals(conn, ts_code, track_date, signals):
    """检查持仓股票是否触发卖点"""

    # 查询持仓中的该股票
    cursor = conn.cursor(pymysql.cursors.DictCursor)
    cursor.execute("""
        SELECT id, stock_name FROM stock_position
        WHERE ts_code = %s AND status = 'holding'
    """, (ts_code,))
    positions = cursor.fetchall()

    if not positions:
        return

    # 对每个持仓记录触发信号
    for pos in positions:
        for signal_type, signal_date in signals.items():
            if signal_date:  # 信号被触发
                # 检查是否已记录
                cursor.execute("""
                    SELECT id FROM stock_alert
                    WHERE position_id = %s AND alert_type = %s AND status = 'pending'
                """, (pos['id'], signal_type))

                if not cursor.fetchone():
                    # 插入新的提示
                    cursor.execute("""
                        INSERT INTO stock_alert
                        (position_id, ts_code, stock_name, alert_type, alert_date, trigger_desc)
                        VALUES (%s, %s, %s, %s, %s, %s)
                    """, (
                        pos['id'],
                        ts_code,
                        pos['stock_name'],
                        signal_type,
                        signal_date,
                        get_alert_desc(signal_type)
                    ))
                    conn.commit()
```

## 9. 技术要点

1. **数据一致性**：买入时同时写入持仓表和交易表，使用事务保证一致性
2. **实时价格**：需要从 `stock_daily` 表获取最新收盘价
3. **盈亏计算**：每次页面加载时重新计算盈亏，不存储在数据库
4. **消息通知**：使用轮询方式检查新提示（每分钟一次）
5. **权限控制**：买入/卖出操作需要登录权限

## 10. 预期效果

1. 用户可以在选股结果页直接买入
2. 系统自动追踪持仓股票的卖点
3. 顶栏实时显示未处理提示
4. 可以查看完整的交易记录和持仓盈亏
5. 避免错过卖点，提升交易效率
